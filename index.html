Project 1

Drive Fusion System(DFS)

인지부터 제어까지 연결한 자율주행 프로젝트

프로젝트 개요

CAN 통신 기반 분산 전장 제어 시스템을 구축하고, 비전 센서를 활용한 장애물 인식·회피 기능을 결합하여 안정적인 자율주행 플랫폼을 구현하는 것을 목표로 했습니다.

시스템 구성

전장 시스템 센서 데이터 차량 구동부 차량 계기판

주요 기능

  1. CAN 통신 기반 ECU 간 데이터 송수신
  2. RTOS 기반 초음파 센서 장애물 감지 및 급정거 기능
  3. 비전 센서 기반 차선 인식 알고리즘을 활용한 주행 제어 기능
  4. 통신 오류 및 센서 이상 상황 대응 Fail-Safe 기능

개인 기여도

강화학습 기반 End-to-End 자율주행은 정적 장애물 환경에서 성과를 확인했지만, 동적 장애물까지 안정화하지 못했습니다.

문제와 해결

01

RTOS 기반 급정거

문제

센서 처리와 모터 제어가 순차 실행되어 정지 반응이 지연되는 문제가 있었습니다.

해결

RTOS 우선순위 태스크를 적용해 장애물 감지 시 즉시 정지하도록 구현했습니다.

02

Fail-Safe 구현

문제

보안사업장 환경으로 인해 Wi-Fi 연결이 불안정하여 제어 신호가 끊기는 현상을 경험했습니다.

해결

통신 신호가 500ms 이상 수신되지 않으면 비정상 상태로 판단하고, 모터 출력을 차단해 차량이 자동 정지하도록 Fail-Safe 로직을 구현했습니다.

03

차선 주행 정렬 알고리즘

문제

카메라 기반 차선 인식만으로는 차량의 안정적인 차선 중앙 유지가 어려웠습니다.

해결

차선 중심 오차를 계산하고 PID 제어로 조향을 보정했습니다.

프로젝트 성과 및 결과

  • 차선 유지 주행 비전 카메라로 차선을 실시간 인식하고 PID 제어로 중심 오차를 보정했습니다.
  • 장애물 감지 및 정지 초음파 센서와 비전 센서로 주행 중 장애물을 감지하고 즉시 정지하도록 구현했습니다.
  • 통합 실주행 검증 차선 유지, 장애물 감지, 정지 기능을 통합한 Drive Fusion System을 완성하고 실주행 테스트를 수행했습니다.

프로젝트 회고

  1. RTOS 태스크를 분리하고 우선순위를 설정하며 단일 루프보다 정지 명령을 안정적으로 처리할 수 있음을 배웠습니다.
  2. CAN 통신을 통해 제어부와 센서부 데이터를 주고받으며 차량 내부 전장 시스템에서 안정적인 통신 구조가 중요하다는 점을 배웠습니다.
  3. STM32, ESP32와 각종 센서 데이터시트를 참고하여 회로와 펌웨어를 설계하며 부품별 전압, 통신 방식, 신호 특성을 이해하는 것이 중요하다는 점을 느꼈습니다.
  4. 강화학습 기반 End-to-End 자율주행을 마무리하지 못한 점은 아쉬움으로 남았습니다.

Project 1.1

DFS 강화학습 기반 End-to-End 자율주행 회고

비전 센서만으로 정적·동적 장애물을 인지하고 정지 및 회피 기동을 수행하는 자율주행 시스템 구현 시도

Success

Train Environment: 정적 장애물

정적인 장애물만 있는 환경에서 회피 주행에 성공했습니다.

Fail

Train Environment: 정적 + 동적 장애물

정적·동적 장애물이 있는 환경에서는 정지 및 회피 주행에 실패했습니다.

Reward 설계

  • 목표 추종 보상 장애물을 회피하고 체크포인트에 도달하면 최종 보상
  • 안전 거리 보상 장애물과 일정 거리 안에서 속도를 줄이거나 정지하면 보상
  • 5방향 회피 보상 전방을 5구역으로 나누어 더 넓은 공간 방향으로 조향하면 보상

Penalty 설계

  • 장애물 근처 과속 패널티 장애물 인식 후 일정 거리 안에서 감속 또는 정지하지 않으면 감점
  • 충돌 패널티 장애물과 접촉 시 큰 감점
  • 불필요한 정지/충돌 패널티 장애물이 없을 때 과한 감속과 조향에 감점

결과

  • 약 50시간 PPO 학습을 통해 체크포인트 사이의 고정 장애물을 회피하며 주행하는 결과를 확인했습니다.
  • 전방을 Left, Left-Center, Center, Right-Center, Right의 5방향으로 나누어 관측하며 막힌 경로와 여유 공간을 판단하게 한 점이 회피 주행에 중요한 역할을 했습니다.
  • 동적 장애물이 추가되면서 policy loss와 episode reward 등의 평가지표가 불안정하게 변동했습니다.
  • 실제 GUI 확인 결과 움직이는 장애물뿐 아니라 고정 장애물에 대해서도 회피하거나 정지하는 동작이 안정적으로 나타나지 않았습니다.

실패 원인

  1. 동적 장애물의 속도, 방향, 이전 위치 정보가 관측값에 부족해 다음 위치를 예측하기 어려웠습니다.
  2. 현재 정책은 이전 프레임을 기억하지 않는 feed-forward PPO 구조이기에 시간에 따라 변하는 장애물 움직임을 안정적으로 학습하기 어려웠습니다.
  3. 감속·정지 보상과 회피 조향 보상이 함께 작용하면서 정책이 정지와 회피 중 어떤 행동을 선택해야 하는지 불안정했습니다.

향후 개선 방향

  1. 움직임 정보 추출 Depth 기반 Optical Flow로 동적 장애물의 이동 방향과 속도 변화 추정
  2. 정지와 회피 행동 분리 학습 정지, 감속, 회피 조향을 상황별 행동으로 분리하도록 보상 구조 재설계
  3. 시계열 기반 정책 구조 개선 이전 프레임의 움직임 흐름을 기억하도록 LSTM/RNN 기반 정책 구조 적용

End-to-End 자율주행 회고

  1. 프로젝트 이후 관련 논문을 찾아보며 모방학습을 활용해 초기 주행 정책을 학습시키는 방식도 효과적일 수 있음을 알게 되었습니다.
  2. 정적 장애물 환경에서 얻은 학습 결과를 동적 장애물 환경으로 확장하기 위해 전이 학습을 적용해보지 못한 점이 아쉬움으로 남았습니다.
  3. End-to-End 자율주행에서는 보상 설계뿐만 아니라 관측 정보와 정책 구조를 함께 설계해야 안정적인 주행으로 이어진다는 점을 배웠습니다.
  4. 강화학습은 단순히 보상을 높이는 것이 아니라 어떤 행동을 유도하고 억제할지 설계하는 과정이 중요하다는 점을 배웠습니다.

디지털 트윈으로 연결한 스마트 물류 공장, 멈추지 않는 흐름

자동 물류 환경에서 컨베이어를 통해 나온 물류를 로봇팔이 AGV에 적재하고, AGV가 지정된 목적지로 이동하며, 이 전체 과정을 실시간으로 모니터링할 수 있는 스마트 물류 자동화 시스템을 구현하는 것을 목표로 했습니다.

Robot Arm UR20 Gripper Camera Conveyor AGV Dashboard

  1. 컨베이어, 로봇팔, AGV의 실시간 상태 모니터링
  2. 로봇팔을 이용한 물류 박스 집기 및 적재 시뮬레이션
  3. AGV의 목적지 이동 및 물류 운반 상태 시각화

로봇 상태, 적재 중량, 이동 경로를 실시간 모니터링하며 이상 발생 시 자동 정지 제어를 수행합니다.

공장 내부 컨베이어 벨트에서 이송되는 3종 크기의 물류를 로봇팔이 자동 인식하여 AGV에 적재한 후 지정 위치로 이동시킵니다.

문제와 해결

01

PPO 알고리즘 기반 로봇팔 강화학습

문제

박스의 크기가 3가지로 나뉘고 위치가 랜덤하게 변해 규칙 기반 제어만으로는 안정적인 파지가 어려웠습니다.

해결

PPO 기반 패널티 및 보상 함수를 설계하여 약 100시간 학습했고, 박스를 안정적으로 파지하도록 구현했습니다.

02

아핀 변환

문제

그리퍼 좌표계와 카메라 좌표계가 달라 객체 중심 추적 시 위치 오차가 발생했습니다.

해결

카메라 좌표와 그리퍼 좌표의 기준점을 매칭한 뒤, 아핀 변환을 통해 객체 위치를 그리퍼 작업 좌표로 변환했습니다.

03

Beam Search

문제

단순 배치 방식은 공간 활용도가 낮아 이후 박스 적재가 제한되는 문제가 있었습니다.

해결

Beam Search 알고리즘을 적용해 여러 적재 후보를 탐색하고, 공간 활용도와 안정성을 고려하여 최적의 적재 위치를 선택했습니다.

프로젝트 성과 및 결과

  • 정렬 오차 개선 목표 박스 중심 정렬 오차를 11.6mm에서 4.1mm로 줄여 약 64.7% 개선했습니다.
  • 공간 활용도 개선 Beam Search 알고리즘을 적용하여 공간 활용도를 23%에서 65.6%까지 개선했습니다.
  • WebSocket 기반 로봇 상태 통신 로봇팔의 동작 상태를 관제 대시보드에 실시간으로 표시했습니다.

프로젝트 회고

  1. 강화학습 기반 Pick & Place 알고리즘을 설계하며 이동 경로, 관절 움직임, End-effector 위치 오차를 고려한 보상 설계의 중요성을 배웠습니다.
  2. 좌표계 기준과 변환 방식을 직접 적용하면서 카메라 좌표계와 그리퍼 작업 좌표계를 정확히 이해하는 것이 목표 위치 도달에 중요하다는 점을 느꼈습니다.
  3. 초기에는 컨베이어 속도를 낮춰 안정성을 확보했지만 실제 환경의 빠른 속도까지 확장하지 못한 점은 아쉬움으로 남았습니다.
  4. 이번 프로젝트를 통해 강화학습뿐 아니라 환경 설계, 단계적 학습 전략, WebSocket 기반 관제 연동의 중요성을 경험했습니다.

Project 3

WEDGE

브라우저 실행부터 UX 진단까지 자동화하는 AI Agent 서비스

프로젝트 개요

WEDGE는 웹사이트의 전환 흐름을 실제 브라우저에서 자동 실행하고, 사용자가 이탈할 수 있는 UX 문제를 근거와 함께 분석해 개선 방향을 제안하는 서비스입니다.

서비스 흐름

URL 입력 사이트 자동 탐색 시나리오 실행 UX 피드백 제공

흐름 상세

  1. 사용자가 분석할 웹사이트 URL을 입력합니다.
  2. WEDGE가 실제 브라우저에서 사이트 구조와 주요 전환 흐름을 탐색합니다.
  3. 선택한 시나리오에 따라 Playwright가 화면을 이동하며 사용자 행동을 재현합니다.
  4. 화면 캡처, DOM, 네트워크, 콘솔 로그를 기반으로 UX 이탈 요인을 분석하고 개선 방향을 제공합니다.

주요 기능
